📋 Section 1 — Project Identity & Stack
Build a full-stack NGO web application for "Anokha Welfare Foundation (AWF)".

BRAND IDENTITY:
- Name: Anokha Welfare Foundation (AWF)
- Tagline: "Empowering Those in Need. Trusted for Truth."
- Philosophy: "Be truth, we are for truth soul"
- Color Palette: Gold (#C87000, #B8860B), Orange (#E05A00, #FF6B00), 
  Off-white (#FAF7F0), Deep Brown (#3B2000) for text
- Logo: AWF monogram with gold metallic treatment
- Tone: Professional, trustworthy, warm, humanitarian

TECH STACK:
- Frontend: React + Next.js (App Router)
- Styling: Tailwind CSS with custom gold/orange theme tokens
- Backend: Node.js / Next.js API routes
- Database: PostgreSQL via Prisma ORM
- Auth: NextAuth.js with role-based access (Admin, Member, Staff)
- Payments: Razorpay SDK (UPI, QR Code, Net Banking, Cards)
- File Storage: AWS S3 or Cloudinary for ID document uploads
- Email: Nodemailer + SMTP for receipts and OTPs
- SMS: Twilio or MSG91 for Indian mobile OTPs
- Hosting: Vercel (frontend) + Railway/Supabase (database)

👥 Section 3 — Membership System
MEMBERSHIP TIERS (3 levels, displayed as pricing cards):

1. GENERAL MEMBER (Annual)
   - Price: Configurable via admin
   - Benefits: Digital ID Card, Foundation newsletters, 
     project updates
   - Badge color: Silver/Gray
   - Razorpay Plan ID: general_annual

2. EXECUTIVE MEMBER (Annual, Renewable)
   - Price: Configurable via admin  
   - Benefits: All General benefits + Priority Emergency Help 
     access + Official Certification letter
   - Badge color: Gold (#C87000)
   - Razorpay Plan ID: executive_annual

3. LIFETIME MEMBER (Permanent)
   - Price: Configurable via admin
   - Benefits: All Executive benefits + Special Honourarium 
     eligibility + Permanent voting rights in the trust
   - Badge color: Deep Gold / Royal
   - Razorpay Plan ID: lifetime_permanent

MEMBERSHIP REGISTRATION FORM:
Multi-step wizard with progress bar (gold color):

Step 1 — Personal Details:
  - Full Name (required)
  - Date of Birth
  - Gender
  - Mobile Number (with OTP verification)
  - Email Address
  - Residential Address (State, District, Pin Code)
  - Occupation

Step 2 — Identity Verification:
  - ID Type selector: Aadhaar / PAN / Voter ID
  - Secure file upload (drag-and-drop, preview, max 5MB PDF/JPG)
  - Passport-size photo upload

Step 3 — Bank Details (for honourarium eligibility):
  - Account Holder Name
  - Bank Name
  - Account Number (masked display)
  - IFSC Code
  - UPI ID (optional)

Step 4 — Membership Selection & Payment:
  - Tier comparison cards (highlighted selection)
  - Razorpay checkout button
  - After payment: auto-generate digital ID card (PNG/PDF)
  - Send welcome email with membership certificate

DATABASE SCHEMA (Prisma):
model Member {

💳 Section 4 — Donation Portal
DONATION CATEGORIES (shown as icon cards, Razorpay Secure badge):

1. Emergency Help — for crisis situations (medical, disaster)
2. ID Verified Donation — authenticated donor, tax receipt issued
3. Project Specific Donation — donor chooses a project to fund
   - Sub-projects: Old Age Home, Education for Children, 
     Widow Support, Health & Hospitals, Water Treatment, 
     Orphanage, Self-Help Groups

DONATION FLOW:
1. User selects donation type
2. Enters amount (preset buttons: ₹500, ₹1000, ₹2000, ₹5000 + custom)
3. Optional: donor name, PAN (for 80G tax exemption), message
4. Razorpay checkout (UPI, Cards, Net Banking, QR Code)
5. Post-payment: 
   - 80G receipt PDF emailed
   - Donation entry in transparency ledger
   - SMS acknowledgment

TRANSPARENCY LEDGER PAGE:
Public page showing total donations received per project with 
progress bar toward annual goal. No individual donor data exposed.

Roles: Super Admin, Accountant, Legal Advisor, Receptionist

Pages:
1. Members Manager — list, approve, suspend, export CSV
2. Donations Manager — all transactions, 80G batch export
3. Projects Manager — CRUD for all service module content
4. Emergency Requests — incoming help requests with priority queue
5. Honorarium Manager — schedule and record payments to 
   religious leaders

8. Global Logistics — import/export tracking for 
   goods/aid distribution

SECURITY REQUIREMENTS:
- All document URLs are signed (expire in 1 hour)
- OTP required for password change
- Admin actions logged with IP + timestamp (audit trail)
- Rate limiting on login endpoints (max 5 attempts)
- HTTPS only, HSTS headers
- CSRF protection on all forms
